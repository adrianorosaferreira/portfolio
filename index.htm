Resume


Este projeto consiste em um currículo online onde é gerado de forma automatizada pelo Javascript a partir do momento que um usuário abra o site. Basicamente ele cria o currículo a partir de um script JS que combina as minhas informações pessoais com os templates HTML e CSS pré-elaboradas. Criado com uma sintaxe básica do Javascript para manipulação de dados, como loops e funções, ao mesmo tempo que utiliza métodos simples de manipulação de jQuery DOM. Além disso é usado uma API do Google Maps para exibir os locais onde vivi, estudei e trabalhei. Sua criação fez parte do curso “Conceitos Básicos de Javascript” desenvolvido pela Udacity.

Arcade Game


Este projeto consiste no jogo de fliperama Frogger. Com base no mecanismo de game loop fornecido pela Udacity, no curso “Javascript Intermediário”, foi criado as entidades do jogo, incluindo personagens jogadores e inimigos e as regras do jogo, como colisões, pontuação e limites de tela. O jogo foi todo desenvolvido explorando o conceito de programação orientada objeto ao qual influencia a arquitetura do aplicativo e fornece otimização de desempenho, como por exemplo a herança e a delegação.

Websites Optimization


Como os aplicativos Web estão cada vez mais interativos e são acessados em uma variedade de dispositivos, há uma variedade de oportunidades nas quais problemas de desempenho possam prejudicar a experiência do usuário. Desta forma, o objetivo deste projeto foi em representar a otimização de um site onde possuísse vários problemas relacionados a desempenho e rendimento. Seu intuito foi em chegar a um índice de classificação boa no PageSpeed e que funciona-se a 60 quadros pode segundo.
Entender os conceitos que embasam a forma como o navegador funciona e interpretar seu código permite que se produza aplicativos de alto desempenho. Desta forma, foram aplicados técnicas que otimização onde considera o caminho de renderização crítica para melhorar a sua performance. Este processo consiste no processo pelo qual o navegador recebe o HTML, CSS e JavaScript e o processo necessário para transformá-los em pixels renderizados. Além disso, foi avaliado e aplicado correção em uma pagina onde possui uma aplicação Web em execução contínua que exige um framerate consistente para produzir a melhor experiência para o usuário.

Neighborhood Map


Esta página foi desenvolvida em página única para apresentar um mapa da região de Florianópolis com alguns locais em destaque, adicionando recursos, dados de terceiros sobre esses locais e diversas formas de navegar pelo conteúdo. O intuito foi exemplificar a utilização de APIs de terceiros (Google Maps e Foursquare) além da utilização de biblioteca/frameworks de UI (Knockout). Este tipo de projeto se caracteriza como sendo complexo e com uma quantidade grande de pontos de dados que pode ser difícil de se gerenciar. Para tanto, existem diversas estruturas, bibliotecas e APIs para ajudar no seu processo de gerenciamento.
Algumas técnicas/implementação que podem ser avaliadas são:

  • Visualização em tempo real dos marcadores sendo filtrados pelo campo de entrada, bem como a lista de lugares
  • Uso de APIs de terceiros para adicionar um recurso que adiciona informações ao clicar em um marcador do mapa ou em uma linha de visualização da lista
  • Utilização da programação assíncrona
  • Uso de animação para fazer o marcador se mover quando ele é selecionado na lista ou quando clicado
  • Interface intuitiva
  • Tratamento de erros
